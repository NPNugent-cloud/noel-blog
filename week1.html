Week 1 — Rebuilding From the Ground Up

I finally admitted something to myself this week: you can’t build a tech career on shaky foundations.

If you spend any time online looking into “tech careers,” you know how chaotic it can be. Every YouTube video claims to reveal the fastest path into tech — cybersecurity, data analysis, software engineering, cloud, AI, helpdesk, DevOps… the list goes on. For a long time, I got caught in that noise. Jumping between tutorials, playlists, and roadmaps, never truly committing to one direction. It felt like trying to build a career on top of quicksand — no structure, no clarity, and definitely no foundation.

This year, something finally clicked: If I wanted a real career, I needed a real path — and cloud engineering is where I belong.

Why I Started With Linux

Every cloud engineer, DevOps engineer, or cybersecurity professional begins with the same core skill: Linux. It’s the OS behind servers, cloud instances, containers, and automation pipelines. But here’s what I realised:

Linux isn’t hard. What’s hard is learning it properly.

So instead of rushing through commands, I slowed down. I worked through real scenarios. I broke things on purpose — then fixed them. I documented everything in a clean, structured GitHub repo. This wasn’t just technical practice — it was discipline.

What I Accomplished in Week 1

  • Linux fundamentals: file navigation, users, groups, permissions
  • GitHub workflow: repo setup, clear documentation, proper commits
  • Mindset training: daily routine, reading error messages, consistency

Cutting Through the Noise

This week taught me that you don’t get closer to a career by watching more videos. You get closer by doing the work — slowly, correctly, and consistently.

Progress is never dramatic in the beginning. It’s built one command, one mistake, and one commit at a time.

Looking Ahead to Week 2

Now that the foundations are in place, I’m moving on to:

  • Advanced Linux permissions and archiving
  • Basic networking commands (ping, traceroute, dig, ip)
  • Strengthening GitHub workflow
  • Continuing to build the habits of a junior cloud engineer

Forward is forward. On to Week 2.
